CPU Usage Tracker

A multithreaded per-core CPU usage monitor for Linux that reads statistics directly from /proc/stat and displays real-time utilization percentages. The program uses a producer-consumer architecture with four dedicated threads, mutex-protected shared data structures, and a watchdog system for thread health monitoring.

Features

  • Producer-consumer pipeline with 4 dedicated threads: reader, analyzer, printer, and watchdog
  • Real-time monitoring of individual CPU core utilization from /proc/stat
  • Mutex synchronization for thread-safe access to shared data structures
  • Watchdog thread monitoring thread health with timestamp tracking and heartbeat detection
  • Graceful shutdown via SIGTERM signal handling
  • Dynamic core detection — automatically adapts to the number of CPU cores
  • Unit tests for core calculation functions
  • Zero memory leaks verified with Valgrind (86 allocs, 86 frees)

Technical Details

  • Language: C with POSIX pthreads
  • Build system: Makefile with gcc
  • Thread architecture: Reader parses /proc/stat, Analyzer computes percentages, Printer formats output, Watchdog monitors heartbeats
  • Synchronization: Mutex-protected shared queues between pipeline stages
  • Signal handling: SIGTERM triggers clean thread shutdown and resource deallocation

Build Instructions

git clone https://github.com/ArturKos/cpu_usage_tracker.git
cd cpu_usage_tracker
make
